# RAG 기반 RFQ 생성 컴포넌트 (PoC)

> 영림원 춘계공모전 대상作 — ERP 전표 기반 경량 SRM의 **RAG 기반 RFQ 생성** 컴포넌트를 단독 구현한 PoC

구매요청 정보를 입력하면, 법무가 사전 승인한 표준계약 조항을 Vector DB에서 검색해 **출처가 명시된 RFQ(견적요청서) 초안**을 생성합니다. 핵심은 RAG로 LLM의 Hallucination을 차단해, 계약 문구가 지어내지지 않도록 보장하는 것입니다.

## 문제 정의 (AS-IS → TO-BE)

| | 기존 방식 (AS-IS) | 제안 방식 (TO-BE) |
|---|---|---|
| 작업 주체 | 구매 담당자가 직접 | RAG 파이프라인이 자동 |
| 조항 확보 | 표준계약서 파일을 일일이 뒤져 복사 | Vector DB에서 관련 조항 자동 검색 |
| 출처 | 수기로 확인, 누락 위험 | 사용된 표준조항을 자동 명시 |
| 위험 | 잘못된 조항 인용·문구 변형 | 근거 없는 내용은 생성 차단 |

구매 담당자가 표준계약서를 뒤져 RFQ를 수작성하던 흐름을, **요청 정보 입력 → 관련 표준조항 검색 → 출처 달린 초안 즉시 생성**으로 대체합니다.

---

## 핵심 특징

- **출처 기반 생성** — 생성된 모든 거래조건에 근거가 된 표준조항(`제N조`)을 함께 표시
- **Hallucination 차단** — 검색된 근거에 없는 내용은 LLM이 생성하지 않도록 프롬프트로 통제
- **입력값 무결성 보장** — 사용자가 입력한 품목·수량·납기는 LLM을 거치지 않고 코드가 직접 조립해, 날짜·수량이 변형되는 문제를 원천 차단
- **조문 단위 검색** — 표준계약서를 `제N조(제목)` 단위로 분할하고 메타데이터를 부여해 정확한 출처 추적

---

## RFQ 문서 구조

생성되는 RFQ는 3개 파트로 구성됩니다.

| 파트 | 내용 | 생성 방식 |
|---|---|---|
| ① 구매품목 정보 | 품목·수량·납기·예산 | 사용자 입력값을 코드가 그대로 삽입 |
| ② 거래조건 | 대금지급·품질보증·하자담보·지체상금 등 | RAG 검색 + LLM 생성 (출처 명시) |
| ③ 공급사 기재란 | 단가·견적·납품가능일 등 | 빈칸 (공급사가 작성, LLM 생성 금지) |

이 분리가 설계의 핵심입니다. **틀리면 안 되는 값(①)은 LLM에 보여주지 않고, 생성이 필요한 부분(②)만 RAG로 처리**합니다.

## 아키텍처

```
[구매요청 입력]
  품목 · 수량 · 납기 · 예산
        │
        ├─────────────────────────────┐
        │ (①은 LLM 우회, 코드가 직접 조립)│
        ▼                             │
[표준조항 검색]                         │
  고정 주제어로 Vector DB 질의           │
  (대금지급·납기·품질보증·하자담보·지체상금) │
        │                             │
        ▼                             │
[거래조건 생성]                         │
  검색된 조항 근거 + LLM (Hallucination 차단)│
  → 거래조건 텍스트 + 출처               │
        │                             │
        ▼                             ▼
        └──────────[RFQ 초안 조립]──────┘
                  ① + ② + ③ 빈칸
```

> **참고:** 현재 구조는 정해진 순서대로 1회 흐르는 **RAG 파이프라인**입니다. 결정론적 상태 그래프(에이전틱 워크플로우) 구성은 LangGraph 기반의 확장 작업으로 별도 진행합니다.

## 코퍼스

- **문서:** 표준하도급계약서 (범용 제조업 분야, 2024.12.24 제정 — 공개 표준계약서 양식)
- **처리:** `제N조(제목)` 정규식으로 조문 단위 분할 후 `article` 메타데이터 부여
- **저장:** Chroma에 벡터로 영구 저장, 검색 시 출처 조문 자동 추적

조문 단위 분할을 문자 단위 분할보다 **먼저** 수행해, 메타데이터가 각 청크까지 전파되도록 했습니다.

## 실행 방법

```bash
# 1. 가상환경 활성화 (Windows PowerShell)
python -m venv venv
.\venv\Scripts\Activate.ps1

# 2. 의존성 설치
pip install langchain langchain-google-genai langchain-chroma langchain-community docx2txt python-dotenv streamlit

# 3. API 키 설정 (.env 파일)
# GOOGLE_API_KEY=your_key_here

# 4. 벡터 DB 구축 (최초 1회)
python rag_langchain.py

# 5. 검색·생성 CLI 실행
python search_cli.py
```

> `.env` 파일에 `GOOGLE_API_KEY`가 필요합니다. (Gemini API 무료 티어 사용)

## 범위

이 PoC는 영림원 SRM의 여러 컴포넌트 중 **RAG 기반 RFQ 생성** 하나를 단독 구현한 것입니다. PR Webhook 연동, 신뢰도 필터(XGBoost+SHAP), 이메일 자동 발송, TCO 분석, 유사품목 병합, ERP 실연동 등 나머지 컴포넌트는 본 PoC 범위에서 제외했습니다.
